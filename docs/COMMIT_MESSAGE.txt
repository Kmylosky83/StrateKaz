fix(programaciones): Corregir problema de timezone en campos DateField

PROBLEMA:
- Al reprogramar una recolección, la fecha seleccionada (ej: 2024-12-01)
  se guardaba con un día menos (2024-11-30)
- Usuario en Bogotá (UTC-5) experimentaba pérdida de 1 día

CAUSA RAÍZ:
- Django con USE_TZ=True convierte fechas UTC a timezone local
- Input "2024-12-01T00:00:00Z" (medianoche UTC) se convertía a
  "2024-11-30 19:00:00" en Bogotá (UTC-5)
- Al extraer la fecha del datetime convertido: 2024-11-30

SOLUCIÓN:
- Creado NaiveDateField custom que ignora timezone para campos de solo fecha
- Para DateField (sin hora), no debería haber conversión de timezone
- Mantiene USE_TZ=True para DateTimeFields (necesarios para timestamps)

CAMBIOS:
- nuevo: apps/core/fields.py - NaiveDateField custom field
- modificado: apps/programaciones/serializers.py
  * ReprogramarSerializer.fecha_reprogramada usa NaiveDateField
