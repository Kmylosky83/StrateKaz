## Bugs descubiertos en validación manual (browser)

Descubiertos post-push mientras se navegaba el tenant_demo para el smoke
pre-deploy. Son **bloqueantes del objetivo operativo MP**.

### 1. RBAC sistémico roto en frontend
**Hallazgo:** `H-S8-rbac-botones-sin-check` (CRÍTICA).

Los botones de acción (eliminar, editar, crear) se renderizan sin consultar
`permission_codes` del usuario. Ejemplo reproducible: super admin desmarca
permiso `delete` para un Cargo, usuario con ese Cargo logoutea + hard
refresh + login, entra a `/catalogo-productos/productos`, y el botón
"Eliminar" sigue visible y funcional.

Backend correctamente rechaza con 403 si se llama al endpoint, pero la UI
no respeta el permiso → UX confusa + viola principio least-privilege.

**Probable sistémico** en todos los módulos LIVE (confirmado al menos en
Catálogo de Productos).

### 2. Modal crear proveedor con UX rota + bug submit
**Hallazgo:** `H-S8-modal-proveedor-ux-rota` (ALTA).

5 problemas identificados:
1. Tipos de MP como checkboxes planos (no escala con >10 ítems).
2. Lista no se re-fetcha si se crea MP nueva en otra pantalla.
3. Multi-select no visible aunque backend lo soporta.
4. Formulario no dinámico por tipo de proveedor.
5. **🚨 Submit falla — no crea proveedor** (bug funcional).

Bloquea 33% del objetivo operativo MP: "crear proveedor + asignar MP +
asignar precio".

## Lecciones aprendidas

1. **Los tests bloqueantes de CI no cubren bugs de UI/UX.** CI passed + `tsc --noEmit` + `eslint` verdes NO significa "producto funcional". La validación manual pre-deploy sigue siendo crítica y debería ser un paso explícito del protocolo antes del push.

2. **Permisos RBAC necesitan testing automático con cargos restringidos.** Hoy los tests usan `create_superuser` por defecto, por lo que nunca se prueba el camino "usuario con permisos limitados → botón escondido". Próximos tests deben simular ese escenario.

3. **Modales complejos requieren auditoría UX dedicada.** "Feature complete" no equivale a "UX aceptable". Estándares de mercado (Linear, Vercel, Stripe, Notion) deberían ser baseline de comparación explícito, no opcional.

4. **`pytest-timeout` es infra crítica.** Sin timeout, un test que cuelga detiene el CI indefinidamente (run de esta sesión: 155.7 min). Agregar `--timeout=120` global.

5. **"LIVE es la única verdad" se cumple cuando el código NO importa de NO-LIVE.** Cerrado en S8 con commit `e51f8b56` (eliminación del mixin `ResumenRevisionMixin`). Deuda análoga registrada como hallazgo para futuras activaciones (H-S8-ct-disperso, H-S8-proveedores-ubicacion-incorrecta).

6. **Migraciones destructivas deben ser idempotentes desde el primer push.** S7 dejó la migración 0004 no-idempotente (cleanup SQL manual en tenant_demo rompió CI en fresh schema). Regla nueva: toda migración con `RemoveField`/`DeleteModel`/`DROP CONSTRAINT` debe usar `SeparateDatabaseAndState` + `DROP ... IF EXISTS` como default.
